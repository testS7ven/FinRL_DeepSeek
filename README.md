## What this repo is

This project trains stock trading agents on 84 NASDAQ stocks using reinforcement learning, with daily news sentiment and risk scores generated by DeepSeek-V3.

The repo contains two layers:

- **Baseline** (`data/`, `envs/`, `training/`, `evaluation/`) — the original four agents from the paper: PPO, CPPO, PPO-DeepSeek, CPPO-DeepSeek.
- **Risk-First extension** (`risk_first/`) — a cleaner, single-file-per-module reimplementation that adds three new modules on top of CPPO-DeepSeek, developed for the FinRL Contest 2025.

## Risk-First extension (`risk_first/`) — original contribution

This is the research contribution developed for the FinRL Contest 2026, Task 1 (AI for Finance — PGE5 2025/2026 at Aivancity). It extends CPPO-DeepSeek with three modules that force the agent to act on LLM risk signals, not just observe them.

| Module | What it does |
| ------ | ------------ |
| Confidence filter | Calls the LLM three times, weights signals by agreement — uncertain scores revert to neutral |
| Reward shaping | Penalises large long positions proportionally to the current risk signal |
| Circuit breaker | Deterministic hard filter: blocks buys and trims positions when risk ≥ 4 and sentiment ≤ 2 |

The key finding from the ablation study: feeding raw LLM scores to PPO (config B) produces the same result as PPO alone (config A). The signal only matters when the architecture is designed to enforce it. The full Risk-First model (config F) reaches a Sharpe ratio of 0.784 and a max drawdown of −56.09%, versus 0.726 and −58.37% for the baseline.

Full documentation, setup instructions, and results: [`risk_first/README.md`](risk_first/README.md)

## Baseline agents

| Agent         | Training script                   | Environment                         | LLM signals      |
| ------------- | --------------------------------- | ----------------------------------- | ---------------- |
| PPO           | `training/train_ppo.py`           | `envs/env_stocktrading.py`          | None             |
| CPPO          | `training/train_cppo.py`          | `envs/env_stocktrading.py`          | None             |
| PPO-DeepSeek  | `training/train_ppo_llm.py`       | `envs/env_stocktrading_llm.py`      | Sentiment        |
| CPPO-DeepSeek | `training/train_cppo_llm_risk.py` | `envs/env_stocktrading_llm_risk.py` | Sentiment + Risk |

Training requires a Linux server with MPI. Example for PPO:

```bash
nohup mpirun --allow-run-as-root -np 8 python training/train_ppo.py > logs/output_ppo.log 2>&1 &
```

Monitor these values in the log during training:

- `AverageEpRet` — should increase over epochs
- `KL` — should stay below `1.5 × target_kl`
- `ClipFrac` — high values indicate an unstable policy

## Data

All data is available on HuggingFace. No local preprocessing needed.

| Dataset                    | Contents                                               | Link                                                                              |
| -------------------------- | ------------------------------------------------------ | --------------------------------------------------------------------------------- |
| `benstaf/nasdaq_2013_2023` | Prices, indicators, DeepSeek-V3 signals (train + test) | [HuggingFace](https://huggingface.co/datasets/benstaf/nasdaq_2013_2023/tree/main) |
| `Zihan1004/FNSPID`         | Raw financial news (15M articles, 1999–2023)           | [HuggingFace](https://huggingface.co/datasets/Zihan1004/FNSPID)                   |
| `benstaf/Trading_agents`   | Pre-trained agent weights                              | [HuggingFace](https://huggingface.co/benstaf/Trading_agents/tree/main)            |

To regenerate LLM signals from scratch, run `data/llm_signals/sentiment_deepseek_deepinfra.py` and `data/llm_signals/risk_deepseek_deepinfra.py` with a DeepInfra API key.

## Evaluation metrics

| Metric                   | Description                              |
| ------------------------ | ---------------------------------------- |
| Cumulative Return        | Total return over 2019–2023              |
| Sharpe Ratio             | Annualised return / volatility           |
| Rachev Ratio             | Tail upside / tail downside (CVaR ratio) |
| Outperformance Frequency | % of days beating the QQQ benchmark      |

Contest score = average rank across the four metrics.
